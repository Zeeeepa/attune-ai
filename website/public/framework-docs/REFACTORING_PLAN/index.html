    Large File Refactoring Plan¶

    Created: 2026-02-02 Status: Planning Owner: Engineering Team

    Staged refactoring of 4 large files to improve maintainability, testability, and code organization.

    src/attune/memory/short_term.py 2197 God class with 50+ methods covering multiple concerns 28 highest src/attune/workflows/base.py 2672 Mixed data models and workflow logic 13 high src/attune/core.py 1701 Multiple classes that could be separate modules core-module medium src/attune/orchestration/execution_strategies.py 2112 Large file with multiple strategy classes specialized lower

    Stage 1: memory/short_term.py¶

    Split god class into focused, single-responsibility modules

    src/attune/memory/short_term.py RedisShortTermMemory 50+ CRUD operations (store, retrieve, delete) Caching layer Pattern staging Session management Pub/Sub messaging Redis Streams Timeline management Queue operations

    src/attune/memory/short_term/ Re-exports RedisShortTermMemory for backward compatibility Core CRUD operations (store, retrieve, delete) Cache layer (get_cached, invalidate, ttl) Pattern staging (stage_pattern, get_staged) Session management (create_session, end_session) Pub/Sub messaging (publish, subscribe) Redis Streams (add_to_stream, read_stream) Timeline management (add_event, get_timeline) Queue operations (enqueue, dequeue) src/attune/memory/short_term.py thin facade composing all modules

    Map all methods to concern categories Identify method dependencies and shared state Document public API for backward compatibility Create test coverage report
